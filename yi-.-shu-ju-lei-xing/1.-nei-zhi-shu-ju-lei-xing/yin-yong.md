# 引用

*   创建: 通过声明符"&"

    例:

    ```
    int ival = 1024;
    int &refVal = ival;
    ```

    其中第二句就创建了一个引用, 即refVal
* 注意
  1. 引用除了在创建时无法绑定其他对象, 所以引用在创建时必须进行初始化, 所以像`int &refVal2;`这样的语句是错误的
  2. 引用并非一个对象, 所以不存在地址, 无法用指针指向它, 也无法定义引用的引用
  3. 非const引用只能绑定在对象上, 而不能与字面值或某个表达式的计算结果绑定在一起
  4. 非const引用的类型都要与和其绑定的对象严格匹配
  5. 引用只是为一个已经存在的对象所起的另外一个名字, 对引用所进行的所有操作都是对与之绑定的对象进行的操作
*   指针的引用 虽然无法定义引用的指针, 但指针的引用是存在的 像这样:

    ```
    int ival = 1024;
    int *pi = &ival;
    int *&r = pi;
    ```

    * 上例中, **pi**是一个指向**ival**的指针, **r**是一个与**pi**绑定的引用. 之所以是一个绑定指针的引用, 我们需要先分析一下第三条语句:
      1. 与**r**靠得最近的是"**&**"声明符, 所以不难判断**r**是一个引用
      2. 而这个引用的类型是`int *`, 即指向`int`的指针, 也就是说**r**是一个与**指向int类型的指针**绑定的**引用**
    * 由于**r**是一个与指针**pi**绑定的引用, 说明对**pi**进行的大部分操作, 也可以对**r**进行, 且效果等同于对**pi**进行, 如: `r = &i;`可以将指针**pi**指向另一个对象, `*r`可以得到**pi**指向的对象. 但无法通过**r**创建一个指向指针的指针
