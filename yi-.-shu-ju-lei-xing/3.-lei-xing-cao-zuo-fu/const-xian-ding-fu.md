# const限定符

const限定符当然在C中也有, 这里主要是几个不同点或者说C中没有的特性

* const的引用 可以把引用绑定到const对象上, 就像绑定到其他对象上一样, 称之为**对常量的引用**
  *   正确与错误形式

      ```
      int i = 42;
      const int &r1 = i;      // 正确: 将const引用绑定到一个普通对象
      const int &r2 = 42;     // 正确: r2是一个常量引用
      const int &r3 = r1 * 2; // 正确: r3是一个常量引用
      int &r4 = r1 * 2;       // 错误: r4是一个普通的非常量引用
      ```
  * 从上述形式可以总结出以下几点:
    1. const引用(常量引用)可以绑定到**普通对象**, **常量**, **一般表达式**
    2. 非const引用(非常量引用)只能绑定到**普通对象**, 而不能绑定到**常量**和**表达式**, 准确来说非常量引用只能绑定到左值.
  * 还有几点需要注意:
    1. 不能给常量引用赋值, 如: `r1 = 0;`这种赋值语句是错误的
    2. 对非常量对象的常量引用并不会改变这个非常量对象本身的性质, 依旧是一个可赋值的普通对象. 对非常量对象的常量引用相当于先创建了一个常量属性的**临时量**, 令其等于被引用的非常量对象的值, 然后让这个常量引用和具有常量属性的**临时量**绑定在一起
* 顶层const和底层const 这是一个在C中存在但并未明确定义的概念, 但在C++中进行了定义
  * 这一概念主要是为了描述指针和引用等复合类型而存在的, 对于指针来说, **顶层const**表示指针本身是个常量, **底层const**表示指针所指的对象是一个常量
  * 顶层const可以表示任意的对象是常量, 这一点对任何数据类型都适用, 如算数类型, 类, 指针等. 由于引用并非对象, 所以其不具有顶层const, 对于const引用, 都表示底层const, 即表示引用的对象是常量. 对于常量引用绑定非常量对象的情况, 上一点说了其实是绑定的一个常量**临时量**.
  * 底层const主要用于表示指针和引用等复合类型的基本类型部分有关: 指针: `const int *const p1 = &r;`这个例子中, 第一个const就是底层const, 即表示指针p1所指的对象是一个常量, 而第二个const是一个顶层const, 表示指针本身是一个常量
  * 用途
    1.  对于顶层const, 当执行对象的拷贝操作时, 其不受影响, 如:

        ```
        int i = 10;
        const int j = 20;
        int n1 = i;         // 正确: 正常赋值操作
        n1 = j;             // 正确: 将常量赋值给对象
        const int n2 = i;   // 正确: 为常量对象初始化
        const int n3 = j;   // 正确: 为常量对象初始化
        n2 = i; n3 = j;     // 错误: 常量对象只能初始化时赋值
        ```

        唯一需要注意的是常量对象只能在初始化时被赋予初始值
    2.  对于底层const, 当执行对象的拷贝操作时, 拷入和拷出的对象必须具有相同的底层const资格, 或者两个对象的数据类型必须能够转换, 一般来说, 非常量可以转换成常量, 反之则不行, 如:

        ```
        int i = 10;
        const int j = 20;
        int *p1 = &i;           // p1是一个非底层const
        const int *p2 = &i;     // p2是一个底层const, 进行了转换
        int *p3 = &j;           // 错误: 常量无法转换为非常量
        const int *p4 = p2;     // 正确: 将底层const拷贝给底层const
        const int *p5 = p1;     // 正确: 将非底层const拷贝给底层const, 经过了一次转换
        int *p6 = p2;           // 错误: 将底层const拷贝给非底层const, 无法转换
        ```

        对于引用来说也一样:

        ```
        int i = 10;
        const int j = 20;
        int &r1 = i;            // r1是一个非底层const
        const int &r2 = i;      // r2是一个底层const, 进行了转换
        int &r3 = j;            // 错误: 将常量与非常量引用绑定, 无法转换
        const int &r4 = r2;     // 正确: 将底层const与底层const引用绑定
        const int &r5 = r1;     // 正确: 将非底层const与底层const引用绑定, 经过了依次转换
        int &r6 = r2;           // 错误: 将底层const与非底层const引用绑定, 无法转换
        ```
